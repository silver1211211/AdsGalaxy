# AdsGalaxy Mobile SDKs

Phase 13 reserves this directory for Android, iOS, and cross-platform mobile SDKs.

Current mobile-ready API surfaces:

- `POST /api/v1/ads/request`
- `POST /api/v1/ads/events`
- `POST /api/v1/rewarded/request`
- `POST /api/v1/rewarded/verify`
- `POST /api/v1/rewarded/callback`
- `POST /api/v1/postbacks`

Use application-specific public keys for client-side loading and private keys for server-to-server reward or conversion validation.
